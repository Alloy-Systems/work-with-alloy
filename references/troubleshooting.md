# Troubleshooting

When asked to troubleshoot a problem, first read Alloy technical documentation section `tech docs/troubleshooting.md` and examine the knowledge base for potential causes.

Make every check you can yourself before involving the user. Do not ask for permission to perform checks; just do them. Check system entities, execution logs, related storage/docs, and available connected systems as relevant. Engage the user only after every check you can make yourself is done.

If you can make a fix, do not tell the user to do it. Ask the user for confirmation, then do it yourself.

When telling the user what to do, or reporting what you did, include a quick navigation link to the item or relevant app section.
